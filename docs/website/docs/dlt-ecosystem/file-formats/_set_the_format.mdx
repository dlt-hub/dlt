<span>There are several ways of configuring dlt to use <code>{props.file_type}</code> file format for normalization step and to store your data at the destination:</span>

1. <span>You can set the <code>loader_file_format</code> argument to <code>{props.file_type}</code> in the run command:</span>

<pre language="py">
info = pipeline.run(some_source(), loader_file_format="{props.file_type}")
</pre>


2. You can set the `loader_file_format` in `config.toml` or `secrets.toml`:

<pre language="toml">
[normalize]{'\n'}
loader_file_format="{props.file_type}"
</pre>

3. You can set the `loader_file_format` via environment variable:

<pre language="sh">
export NORMALIZE__LOADER_FILE_FORMAT="{props.file_type}"
</pre>

4. You can set the file type directly in [the resource decorator](../../general-usage/resource#pick-loader-file-format-for-a-particular-resource).

<pre language="py">
@dlt.resource(file_format="{props.file_type}"){'\n'}
def generate_rows(nr):{'\n'}
&nbsp;&nbsp;&nbsp;&nbsp;pass
</pre>
