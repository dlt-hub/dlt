## So is DuckDB full of data about ducks?

Nope, you can put whatever data you want into DuckDB ‚ú®

Many data analysts, data scientists, and developers prefer to work with data on their laptops. DuckDB allows them to start quickly and easily. When working only locally becomes infeasible, they can then turn this local ‚Äúdata pond‚Äù into a data lake, storing their data on object storage like Amazon S3, and continue to [use DuckDB as a query engine on top of the files stored there](https://datamonkeysite.com/2023/02/23/implementing-a-poor-mans-lakehouse-in-azure/).

If you want to better understand why folks are excited about DuckDB, check out [this blog post](https://dlthub.com/docs/blog/duckdb-1M-downloads-users).

## Maybe it‚Äôs called DuckDB because you can use it to create a "data pond" that can grow into a data lake + ducks like water?

Although this is a cool idea, it is still not the reason that it is called DuckDB üåä

Using functionality offered by DuckDB to export the data loaded to it as Parquet files, you can create a small ‚Äúdata pond‚Äù on your local computer. To make it a data lake, you can then add these files to [Google Cloud Storage](https://cloud.google.com/storage), [Amazon S3](https://aws.amazon.com/s3/), etc. And if you want this data lake to always fill with the latest data from the GitHub API, you can [deploy the dlt pipeline](https://dlthub.com/docs/walkthroughs/deploy-a-pipeline/deploy-with-github-actions).

Check this out in the [Colab notebook](https://colab.research.google.com/drive/1BXvma_9R9MX8p_iSvHE4ebg90sUroty2) and [let us know](https://github.com/dlt-hub/dlt/issues/new) if you want some help setting this up.
